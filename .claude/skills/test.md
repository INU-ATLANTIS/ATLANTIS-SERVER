# Test Generation Skill

테스트 코드를 생성하거나 기존 테스트를 분석한다. `/test <파일명 또는 클래스명>`으로 호출.

## 실행 순서

1. 대상 클래스/메서드 파악
2. 테스트 전략 제안 (단위/통합/E2E)
3. 테스트 케이스 목록 사용자 확인
4. 승인 후 테스트 코드 작성

## 테스트 전략

### Spring Boot 기준
- **Controller**: MockMvc 사용, HTTP 상태코드 & 응답 바디 검증
- **Service**: Mock 주입, 비즈니스 로직 단위 테스트
- **Repository**: @DataJpaTest, 실제 DB 쿼리 검증

### 테스트 케이스 구성
```
✅ 정상 케이스
  - 기본 성공 시나리오

❌ 예외 케이스
  - 입력값 누락/잘못된 형식
  - 존재하지 않는 리소스
  - 권한 없는 접근

🔲 경계값 케이스
  - 최솟값 / 최댓값
  - 빈 목록 / null
```

## 규칙

- 테스트 파일 생성 전 케이스 목록 사용자 확인
- given / when / then 구조 준수
- 테스트 메서드명 한글 허용 (`@DisplayName` 활용)
- 기존 테스트 파일 수정 시 반드시 허락
