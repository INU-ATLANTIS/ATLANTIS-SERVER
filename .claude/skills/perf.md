# Performance Review Skill

성능 이슈를 탐지하고 개선 방안을 제시한다. `/perf <파일명 또는 범위>`로 호출.

## 실행 순서

1. 대상 코드 분석
2. 성능 이슈 탐지 및 심각도 분류
3. 개선 방안 제시 후 사용자 승인
4. 승인된 항목만 수정

## 점검 항목

### DB / JPA
- [ ] N+1 쿼리 문제 (Lazy Loading 남용)
- [ ] 인덱스 누락 가능성
- [ ] 불필요한 전체 컬럼 조회 (SELECT *)
- [ ] 페이지네이션 없는 목록 조회
- [ ] 트랜잭션 범위 과도하게 큰 경우

### 애플리케이션
- [ ] 반복문 안에서 DB 조회
- [ ] 불필요한 객체 생성
- [ ] 동기 처리로 인한 블로킹
- [ ] 캐싱 적용 가능한 반복 연산

### API
- [ ] 응답 데이터 크기 (불필요한 필드 포함)
- [ ] 압축 미적용

## 출력 형식

```
⚡ 성능 점검 결과

🔴 High Impact (N건)
  [파일:라인] 문제 설명 → 예상 개선 효과

🟡 Medium Impact (N건)
  [파일:라인] 문제 설명 → 예상 개선 효과

🔵 Low Impact (N건)
  [파일:라인] 제안

수정 진행할까요?
```

## 규칙

- 수정 전 반드시 사용자 승인
- 성능 수치 추정 시 "예상" 임을 명시
- DB 스키마 변경(인덱스 추가 등)은 별도 확인
