# Code Review Skill

코드 변경사항을 전문적으로 리뷰한다. `/review` 또는 `/review <파일명>`으로 호출.

## 실행 순서

1. 대상 파일이 지정되면 해당 파일만, 없으면 `git diff` 전체 리뷰
2. 아래 항목 순서대로 분석
3. 결과 보고 후 수정 여부 사용자에게 확인

## 리뷰 항목

### 🔴 Critical (반드시 수정)
- 보안 취약점 (SQL Injection, XSS, 인증 누락 등)
- 데이터 유실 가능성
- NPE / 런타임 에러 가능성
- 민감 정보 하드코딩 (API 키, 비밀번호 등)

### 🟡 Warning (수정 권장)
- 중복 코드
- 불필요한 복잡도
- 예외 처리 누락
- 성능 이슈 (N+1 쿼리, 불필요한 루프 등)

### 🔵 Suggestion (선택)
- 네이밍 개선
- 가독성 향상
- 테스트 추가 권장

## 출력 형식

```
🔍 코드 리뷰 결과

🔴 Critical (N건)
  [파일:라인] 문제 설명 → 수정 방법

🟡 Warning (N건)
  [파일:라인] 문제 설명 → 수정 방법

🔵 Suggestion (N건)
  [파일:라인] 제안 내용

수정이 필요한 항목을 고칠까요?
```

## 규칙

- Critical 항목은 수정 코드를 직접 제시
- 수정 전 반드시 사용자 승인 받기
- 파일 경로와 라인 번호 항상 명시
