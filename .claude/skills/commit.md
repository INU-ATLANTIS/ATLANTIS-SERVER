# Git Commit Skill

커밋 요청 시 변경사항을 분석하고 사용자 확인 후 커밋한다.

## 실행 순서

1. `git status`와 `git diff` 로 변경사항 전체 파악
2. 변경 내용을 아래 형식으로 사용자에게 보고
3. **삭제 또는 수정 항목이 있으면 반드시 허락 받기**
4. 허락 받은 후 커밋 메시지 제안 → 최종 확인 → 커밋

## 보고 형식

변경사항을 아래처럼 분류해서 보여준다:

```
📋 변경사항 요약

➕ 추가 (N개)
  - 파일명

✏️ 수정 (N개)   ← 허락 필요
  - 파일명: 주요 변경 내용 한 줄 요약

🗑️ 삭제 (N개)   ← 반드시 허락 필요
  - 파일명

커밋할까요?
```

## 엄격한 규칙

- **삭제 파일**이 있으면 무조건 사용자에게 "정말 삭제하시겠습니까?" 재확인
- **수정 파일**이 있으면 주요 변경 내용 요약 후 허락 요청
- 사용자가 명시적으로 "ㅇㅇ", "응", "yes", "커밋해" 등으로 승인해야만 커밋 실행
- 승인 없이 절대 커밋하지 않는다

## 커밋 메시지 형식

Gitmoji + Conventional Commits (CLAUDE.md 참고):
```
<이모지> <type>(<scope>): <한글 설명>
```
