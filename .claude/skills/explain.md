# Explain Skill

코드나 개념을 이해하기 쉽게 설명한다. `/explain <파일명, 함수명, 또는 개념>`으로 호출.

## 실행 순서

1. 대상 코드/개념 파악
2. 사용자 맥락 파악 (필요 시 질문)
3. 계층적으로 설명 (전체 → 세부)

## 설명 구조

### 코드 설명 시
```
📌 한 줄 요약
  이 코드가 하는 일

🔄 흐름
  1. 입력: 무엇을 받는지
  2. 처리: 어떻게 처리하는지
  3. 출력: 무엇을 반환하는지

⚙️ 핵심 로직
  중요한 부분 집중 설명

🔗 연관 관계
  다른 클래스/메서드와의 관계
```

### 개념 설명 시
- 한 줄 정의 → 왜 필요한지 → 이 프로젝트에서 어떻게 쓰이는지

## 규칙

- 코드를 직접 수정하지 않는다 (읽기 전용 스킬)
- 전문용어 사용 시 괄호로 간단히 풀이
- 필요하면 간단한 예시 코드 추가
