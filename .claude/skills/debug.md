# Debug Skill

에러 또는 버그를 체계적으로 분석하고 원인을 찾는다. `/debug <에러메시지 또는 파일명>`으로 호출.

## 실행 순서

1. 에러 메시지 또는 증상 파악
2. 관련 파일 및 코드 추적
3. 원인 가설 수립 → 검증
4. 수정 방법 제시 후 사용자 승인

## 분석 프레임워크

### 1단계: 에러 분류
- 컴파일 에러 / 런타임 에러 / 로직 에러 / 환경 문제

### 2단계: 원인 추적
- 스택 트레이스 분석
- 관련 코드 흐름 추적 (Controller → Service → Repository)
- 최근 변경사항(`git log`) 확인

### 3단계: 가설 & 검증
```
🔍 원인 가설
  1순위: [가장 가능성 높은 원인]
  2순위: [차선 가능성]

📍 확인 필요 위치
  - 파일명:라인
```

### 4단계: 수정 제안
- 수정 코드 제시
- 수정 시 발생 가능한 사이드 이펙트 명시
- 수정 전 사용자 승인

## 규칙

- 추측으로 수정하지 않는다. 원인이 불명확하면 추가 정보 요청
- 수정은 반드시 사용자 승인 후 진행
- 수정 범위 최소화 원칙
