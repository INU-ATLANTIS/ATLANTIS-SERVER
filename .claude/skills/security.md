# Security Audit Skill

프로젝트의 보안 취약점을 점검한다. `/security` 또는 `/security <파일명>`으로 호출.

## 실행 순서

1. 대상 범위 확인 (전체 or 특정 파일)
2. 아래 체크리스트 기준으로 순차 점검
3. 발견 항목 심각도별 분류 후 보고
4. 수정 여부 사용자 확인

## 점검 항목

### 인증 & 인가
- [ ] JWT 검증 로직 누락
- [ ] 권한 없는 엔드포인트 접근 가능 여부
- [ ] 토큰 만료 처리
- [ ] SecurityConfig 설정 누락 경로

### 입력 검증
- [ ] SQL Injection 가능성
- [ ] XSS 취약점
- [ ] 파일 업로드 확장자/용량 검증
- [ ] Request DTO @Valid 누락

### 민감 정보
- [ ] 하드코딩된 비밀번호 / API 키
- [ ] 로그에 민감 정보 출력
- [ ] 응답에 불필요한 민감 필드 포함

### 의존성
- [ ] 알려진 취약점이 있는 라이브러리 버전

## 출력 형식

```
🔐 보안 점검 결과

🚨 High (N건) — 즉시 수정 필요
  [파일:라인] 취약점 설명

⚠️ Medium (N건)
  [파일:라인] 취약점 설명

💡 Low (N건)
  [파일:라인] 개선 제안

수정 진행할까요?
```

## 규칙

- High 항목은 수정 코드 직접 제시
- 수정 전 반드시 사용자 승인
- 환경변수 파일(.env, application.yml) 직접 수정 금지
