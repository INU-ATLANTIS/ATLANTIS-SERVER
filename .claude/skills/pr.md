# Pull Request Skill

PR을 생성하거나 기존 PR을 분석한다. `/pr` 또는 `/pr <PR번호>`로 호출.

## PR 생성 시 실행 순서

1. `git log`로 main 브랜치 대비 커밋 히스토리 파악
2. `git diff main...HEAD`로 전체 변경사항 분석
3. PR 제목 및 본문 초안 작성 후 사용자 확인
4. 승인 후 `gh pr create` 실행

## PR 본문 형식

```markdown
## 변경 사항
- 

## 변경 이유
- 

## 테스트 방법
- [ ] 

## 체크리스트
- [ ] 보안 이슈 없음
- [ ] 환경변수 노출 없음
- [ ] 불필요한 파일 미포함
```

## PR 리뷰 시 (`/pr <번호>`)

- 변경 파일 목록 및 규모 파악
- 리뷰 포인트 요약
- `/review` 스킬과 연계하여 코드 품질 분석

## 규칙

- PR 생성 전 제목/본문 반드시 사용자 확인
- base 브랜치 자동 감지 (main / master)
- draft PR 여부 사용자에게 질문
