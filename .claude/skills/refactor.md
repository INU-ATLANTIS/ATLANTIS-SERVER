# Refactor Skill

코드를 안전하게 리팩토링한다. `/refactor <파일명 또는 범위>`로 호출.

## 실행 순서

1. 대상 코드 분석 및 리팩토링 포인트 파악
2. 리팩토링 계획 사용자에게 보고
3. **수정 범위와 영향도 명시 후 승인 받기**
4. 단계별 수정 진행 (한 번에 전부 바꾸지 않음)

## 리팩토링 대상 탐지

### 코드 냄새 (Code Smells)
- 중복 코드 (같은 로직 2곳 이상)
- 긴 메서드 (30줄 초과)
- 과도한 파라미터 (4개 초과)
- 매직 넘버/스트링
- 깊은 중첩 (if/for 3단계 이상)

### Spring Boot 특화
- Service 레이어에 Controller 로직 혼재
- Repository에서 직접 비즈니스 로직 처리
- DTO ↔ Entity 변환 로직 분산

## 출력 형식

```
♻️ 리팩토링 계획

📍 대상: 파일명
🎯 목적: 리팩토링 이유 한 줄
📋 변경 범위:
  - 수정할 내용 1
  - 수정할 내용 2
⚠️ 영향 범위: 이 변경이 영향을 주는 다른 파일

진행할까요?
```

## 규칙

- **기능 동작은 절대 변경하지 않는다**
- 수정 전 반드시 사용자 승인
- 삭제되는 코드가 있으면 이중 확인
- 리팩토링 완료 후 영향 받는 파일 목록 보고
